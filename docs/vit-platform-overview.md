# VIT 平台全貌

## 结论

VIT（V&V-driven Iteration Tower）是面向整车测试与软件测试协同的 V&V 驱动迭代平台。

它不是单一测试执行平台，而是通过整车测试验收反馈、软件测试自动化、世界仿真模型和多 Agent 协同，形成从真实问题到软件回归、从需求分析到持续验证、从仿真环境到质量反馈的闭环。

## 平台目标

VIT 要解决的问题：

- 整车测试任务和结果如何系统化管理。
- 整车测试发现的真实问题如何转化为软件测试资产。
- 软件测试如何从需求分析、策略生成、用例生成、自动化开发到 CI/CT 形成闭环。
- 软件测试所需的人、车、路、站、规则、事件、故障如何通过虚拟世界环境模拟出来。
- 多 Agent 生成的测试资产如何被规范、评审、执行和度量。
- 测试数据、仿真数据和 AI 产物如何治理，避免脏数据、低质量产物和安全合规风险。

## 三大业务域

```text
VIT 平台
├── 整车测试域：验收业务
├── 软件测试域：V&V 自动化 / Harness
└── 软件测试中台：世界仿真模型
```

### 整车测试域：验收业务

验收业务面向整车测试，主要服务于真实车辆、真实场站、真实路线下的测试任务排布和结果反馈。

核心职责：

- 站点/路线测试任务管理。
- 场站测试任务管理。
- 测试任务排布。
- 任务发起。
- 任务状态查询。
- 整车测试结果反馈。
- 真实问题、真实场景、异常记录沉淀。

验收业务不负责软件自动化测试代码，也不承载软件测试 Harness。

### 软件测试域：V&V 自动化

V&V 自动化面向软件测试，是测试开发 Harness 的核心承载域。

核心职责：

- 产品需求分析。
- 测试策略生成。
- 测试用例生成。
- 接口自动化。
- UI 自动化。
- 场景自动化。
- 缺陷分析。
- 测试看板/报告生成。
- 测试建议生成。
- 自动冒烟。
- 自动线上巡检。
- 自动问题回归。
- CI/CT 持续验证。

V&V 自动化的重点不是只跑测试，而是规范软件测试开发、沉淀测试资产、约束 Agent 产物、执行测试任务并输出可度量反馈。

### 软件测试中台：世界仿真模型

世界仿真模型面向软件测试，是测试中台服务。

它负责模拟软件测试所需的虚拟世界环境：

- 基础环境：道路拓扑、市政信息、自然环境。
- 智能实体：乘客、被调度车辆、被调度人员。
- 环境实体：静态实体、动态实体。
- 状态管理：实体状态、状态流转、时间推进。
- 规则引擎：交通规则、事件触发、交互规则。
- 故事仿真：初始化、故事编辑器、故事生成器。
- 场景注册：可复用仿真故事、异常注入模板。
- 仿真轨迹：状态流、事件流、实体轨迹和回放索引。

世界仿真模型通过标准 API 被 V&V 自动化 Harness 调用，支撑场景验证、问题复现、回归测试、批量订单模拟、调度策略验证和异常注入。

## Harness 定位

Harness 位于 `vv-automation/harness/`，是软件测试侧的测试开发与执行工程底座。

它负责：

- 规范测试工具、测试场景、测试用例、测试数据、校验器、报告等测试资产。
- 通过 Schema、模板、脚手架、Lint 和质量门禁约束测试开发行为。
- 通过 adapter 接入 OMS、RAS、移动端、车云、世界仿真模型等系统。
- 通过 fixture 管理测试数据准备、环境上下文、数据隔离和清理。
- 通过 validator 固化订单、车辆、调度、计费、状态机、数据一致性等确定性规则。
- 通过 runtime 统一执行、调度、回放和 CI/CT 接入。
- 通过 observability 采集日志、指标、trace 和诊断信息。
- 通过 evidence 留存测试证据、仿真轨迹、回归证明和问题复现材料。

Harness 不替代整车测试任务管理，也不直接承担验收业务职责。

## Agent、Skill、Memory、Knowledge

```text
agents/     管协作
skills/     管能力
memory/     管长期经验
knowledge/  管业务事实
harness/    管工程约束和执行验证
```

### Agents

`agents/` 存放多 Agent 协同配置。

它包括：

- 角色定义。
- 工作流。
- Prompt 模板。
- 安全和输出策略。
- Agent 评估。
- Agent 与 Skill 的绑定关系。

Agent 负责生成、修改、评审和分析测试资产，但不能绕过 Harness 的契约、质量门禁和执行验证。

### Skills

`skills/` 存放可复用能力包。

典型 Skill：

- 测试设计。
- 测试开发。
- 缺陷分析。
- 仿真故事生成。
- 数据质量分析。
- 报告生成。

Skill 是能力，不绑定单一 Agent。一个 Skill 可以被多个 Agent 复用。

### Memory

`memory/` 存放项目长期记忆。

它沉淀：

- 平台定位。
- 模块边界。
- 架构决策。
- 经验复盘。
- Agent 使用反馈。

Memory 不存放密钥、生产账号、未脱敏用户数据、敏感车辆轨迹或其他不可共享信息。

### Knowledge

`knowledge/` 存放相对稳定的业务事实和测试知识。

它沉淀：

- 订单、车辆、场站、调度、支付、结算规则。
- 订单状态机、车辆状态机、任务状态机。
- 测试策略。
- 风险模型。
- 历史缺陷。
- 数据质量规则。

## Contracts 与 Governance

### Contracts

`contracts/` 存放跨域对象模型、Schema 和接口契约。

核心契约：

- 整车验收反馈到软件测试资产。
- V&V 自动化到世界仿真模型。
- Agent 生成产物。
- 测试资产。
- 遥测和执行证据。

跨域协作优先通过契约进行，避免验收、自动化、仿真、Agent 各自定义一套数据。

### Governance

`governance/` 存放平台治理规则。

治理范围：

- AI 生成代码、用例、报告、建议的审核和准入。
- 敏感数据、脱敏、权限、密钥、生产数据隔离。
- 仿真数据真实性、偏差、污染和可追溯性。
- 发布准入、质量门禁、风险升级和人工复核。

治理原则：

- AI 提升效率，但不能绕过确定性校验。
- 敏感数据和密钥不进仓库。
- 仿真数据必须可追溯来源、生成规则和适用范围。
- 高风险 Agent 产物必须经过人工复核和 Harness 质量门禁。

## 端到端闭环

### 闭环 1：整车问题到软件回归

```text
整车测试任务
  -> 验收业务记录状态和结果
  -> issue-intake 沉淀真实问题
  -> contracts/acceptance-to-vv 转换为回归候选
  -> Agent 生成回归场景和仿真故事
  -> Harness 校验并执行
  -> 测试报告和回归证据沉淀
```

### 闭环 2：需求到自动化验证

```text
产品需求
  -> 需求 Agent 分析业务规则和风险
  -> 测试策略 Agent 生成策略
  -> 用例 Agent 生成场景和用例
  -> 测试开发 Agent 生成工具、fixture、validator
  -> Harness 质量门禁
  -> CI/CT 执行
  -> 报告、缺陷、建议反馈
```

### 闭环 3：软件测试到世界仿真

```text
测试场景
  -> contracts/vv-to-world-sim
  -> 世界仿真模型初始化环境和实体
  -> 注入规则、事件、故障
  -> 推进状态和时间
  -> 输出仿真 trace、事件流、实体轨迹
  -> Harness validator 做确定性校验
  -> evidence 留存回放材料
```

### 闭环 4：Agent 产物治理

```text
Agent 生成资产
  -> contracts/agent-artifacts Schema 校验
  -> governance/ai-output 审核
  -> Harness quality-gates 检查
  -> 单测/示例/Prompt 回归
  -> 通过后进入测试资产库或执行链路
```

## 推荐一级目录

| 目录 | 定位 |
|---|---|
| `vehicle-acceptance/` | 整车测试验收业务域 |
| `vv-automation/` | 软件测试 V&V 自动化域 |
| `world-sim/` | 世界仿真模型中台服务 |
| `agents/` | 多 Agent 协同配置 |
| `skills/` | 可复用 Agent 能力包 |
| `memory/` | 项目长期记忆 |
| `knowledge/` | 业务、测试、数据质量知识 |
| `contracts/` | 跨域对象模型、Schema、接口契约 |
| `governance/` | AI 产物、数据、安全、发布治理 |
| `docs/` | 架构、规范、接口和演进文档 |

## 演进优先级

### P0：先建立确定性底座

- 定义核心测试资产契约。
- 定义 V&V 到世界仿真的最小 API 契约。
- 建立 Harness Schema、模板、脚手架和质量门禁。
- 建立 fixture 数据准备和清理规范。
- 建立 validator 确定性校验器。
- 建立日志、指标、trace 和 evidence 最小规范。
- 建立 Agent 与 Skill 绑定规范。
- 建立 Prompt 回归评估样本。

### P1：打通关键闭环

- 整车验收问题到软件回归候选的转化。
- 世界仿真故事模板库。
- 需求到用例、问题到回归的 Agent 工作流。
- 测试设计、测试开发、仿真故事首批 Skill。
- AI 产物和数据安全审核规则。

### P2：平台化和规模化

- 仿真故事自动生成。
- 大规模 CI/CT 调度。
- 执行证据与质量看板打通。
- Agent 采纳率、修改后通过率、覆盖率提升等指标闭环。
- 脏数据投毒检测和仿真数据质量治理。
