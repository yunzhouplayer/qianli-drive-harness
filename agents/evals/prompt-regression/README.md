# Prompt Regression Eval

## 目标

本目录用于评估 Product、Development、Testing、Review、Critic 五类 Agent 的输出是否稳定、可追溯、可准入。

Prompt 或模型配置变更后，必须用固定样本做回归，避免只凭主观感觉判断输出质量。

## 最小评估集

建议至少沉淀三类样本：

- 乘客端 PRD：登录、站点推荐、订单状态、支付、消息、分享、反馈。
- 整车验收问题：真实道路/场站问题转软件回归资产。
- 仿真故事需求：人、车、路、站、规则、事件、故障注入。

## 指标

| 指标 | 目标 |
|---|---|
| 需求单元重复率 | 小于 5% |
| 需求闭环覆盖率 | 100% |
| 测试点覆盖率 | 大于 95% |
| P0/P1 人工评审标记率 | 100% |
| 澄清项消费率 | 高风险澄清项必须被后续资产引用或标记阻断 |
| Gate 阻断准确性 | 缺少追溯、证据或评审时必须 blocked/warning |
| Critic 缺口召回 | 已知覆盖缺口必须被识别 |

## 回归记录

每次回归建议记录：

- 输入样本 ID。
- 使用的 workflow。
- 使用的 prompts 版本。
- 使用的模型和 provider。
- 需求单元数、测试点数、用例数。
- coverage / duplicate / gate / critic 指标。
- 人工复核结论。
