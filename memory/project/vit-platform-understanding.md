# VIT 平台理解

## 平台定位

VIT（V&V-driven Iteration Tower）是面向整车测试与软件测试协同的 V&V 驱动迭代平台。

它不是单一测试执行平台，而是通过整车测试反馈、软件测试自动化和世界仿真中台形成持续迭代闭环。

## 模块边界

### 验收业务

验收业务面向整车测试，主要服务于整车测试的任务排布和结果反馈。

核心职责：

- 站点/路线测试任务管理。
- 场站测试任务管理。
- 任务发起。
- 状态查询。
- 测试结果反馈。
- 真实问题沉淀。

### V&V 自动化

V&V 自动化面向软件测试，是测试开发 Harness 的核心承载域。

核心职责：

- 产品需求分析。
- 测试策略生成。
- 测试用例生成。
- 接口/UI/场景自动化。
- 缺陷分析。
- 测试看板/报告生成。
- 测试建议生成。
- 自动冒烟。
- 自动线上巡检。
- 自动问题回归。
- CI/CT。

### 世界仿真模型

世界仿真模型面向软件测试，是测试中台服务。

它负责模拟软件测试所需的虚拟世界环境，包括道路、市政、自然环境、乘客、车辆、人员、静态实体、动态实体、状态管理、交通规则、事件触发、交互规则、故障和故事仿真。

## Harness 定位

Harness 位于 V&V 自动化域内，是测试开发与执行工程底座。

它负责规范软件测试开发、测试资产、Agent 产物、执行反馈和质量门禁；同时通过适配器调用世界仿真模型，并消费验收业务沉淀的真实问题与真实场景。

Harness 不替代整车测试任务管理。
